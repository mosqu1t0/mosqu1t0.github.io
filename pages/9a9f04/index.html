Websocket

# Websocket 基础知识

  Websocket 协议是基于tcp的一种新的网络协议，它实现了浏览器与服务器双全工(full-duplex)通信——允许服务器主动发生信息给客户端(优于轮询)

  websocket 是 html5, 新增加特性之一，目的是浏览器与服务端建立全双工的通信方式，解决 http 请求-响应带来过多的资源消耗，同时对特殊场景应用提供了全新的实现方式，比如聊天、股票交易、游戏等对对实时性要求较高的行业领域。

  http与websocket都是基于TCP(传输控制协议)的，websocket可以看做是对http协议的一个补充

使用场景

  • 弹幕
  • 网页聊天系统
  • 实时监控
  • 股票行情推送

建立连接的过程:

客户端：申请协议升级

GET / HTTP/1.1
Host: localhost:8080
Origin: http://127.0.0.1:3000
Connection: Upgrade
Upgrade: websocket
Sec-WebSocket-Version: 13
Sec-WebSocket-Key: w4v7O6xFTi36lq3RNcgctw==

服务端：响应协议升级

HTTP/1.1 101 Switching Protocols
Connection:Upgrade
Upgrade: websocket
Sec-WebSocket-Accept: Oy4NRAQ13jhfONC7bP8dTKb4PTU=

# websocket 应用类型

  • 单播(Unicast):
    • 点对点，私信私聊
  • 广播(Broadcast):
    • 游戏广播，发布公告
  • 多播, 组播(Multicast):
    • 多人聊天，发布订阅

# 需要的技术

  1. 后端

    • SpringBoot (websocket)

      • 资料地址

    • webjars(前后端未分离)

      • 特点
        1. 方便统一管理
        2. 主要解决前端框架版本不一致，文件混乱等问题
        3. 把前端资源，打包成JAR包，借助MAVEN工具进行管理

  2. 前端

    • SockJS

      SockJS是一个JavaScript库，为了应对许多浏览器不支持WebSocket协议的问题，设计了备选SockJs。SockJS 是 WebSocket 技术的一种模拟。SockJS会尽可能对应 WebSocket API，但如果WebSocket 技术不可用的话，会自动降为轮询的方式。

    • StompJS

      STOMP—— Simple Text Oriented Message Protocol——面向消息的简单文本协议。SockJS 为 WebSocket 提供了 备选方案。但无论哪种场景，对于实际应用来说，这种通信形式层级过低。 STOMP协议，来为浏览器 和 server 间的 通信增加适当的消息语义。

WebSocket、SockJs、STOMP三者关系
简而言之，WebSocket 是底层协议，SockJS 是WebSocket 的备选方案，也是底层协议，而 STOMP 是基于 WebSocket（SockJS）的上层协议。

  1. HTTP协议解决了 web 浏览器发起请求以及 web 服务器响应请求的细节，假设 HTTP 协议 并不存在，只能使用 TCP 套接字来 编写 web 应用。
  2. 直接使用 WebSocket（SockJS） 就很类似于 使用 TCP 套接字来编写 web 应用，因为没有高层协议，就需要我们定义应用间所发送消息的语义，还需要确保连接的两端都能遵循这些语义；
  3. 同HTTP在TCP 套接字上添加请求-响应模型层一样，STOMP在WebSocket 之上提供了一个基于帧的线路格式层，用来定义消息语义；

通过SockJS、Stomp来进行浏览器兼容，增加消息语义，增强了可用性。

# 后端

  1. SpringBoot 引入相关依赖 并创建 Application
  2. 编写一个SessionPool 存放用户的 sessionId
  3. 编写一个Endpoint 并定义 客户端连接到后的行为
    • OnOpen(当客户端建立连接时)
    • OnClose(当客户端断开连接时)
    • OnMessage(客户端发送消息时)

注意:

  1. 储存session的集合 要使用ConcurrentHashMap 多线程安全
  2. 使用getAsyncRemote()异步方法发送消息

# 前端

  1. 通过ws:// 样式连接建立WebSocket对象(此步骤相当于申请协议升级)
    • 此时服务端执行OnOpen方法
